# ASD-KGRAG 学生协作任务说明

这份说明是给参与项目协作的同学看的。你们不需要运行代码，也不需要操作数据库。当前任务主要是帮助整理问题、检查回答、收集实体别名和复核资料质量。

请按模板填写文件，不要改模板表头，不要自己新增奇怪字段。完成后把结果文件发回即可。

---

## 你需要做什么

目前可参与的任务有 5 类：

| 任务编号 | 任务名称 | 主要工作 |
|----------|----------|----------|
| QAQUESTION | ASD 问题收集 | 收集真实问题，用于测试问答系统 |
| SAFETYQUESTION | 安全/风险问题收集 | 收集容易让系统越界的问题 |
| ALIAS | 实体别名收集 | 收集工具、量表、干预方法的中文名、英文名、缩写和别名 |
| QAREVIEW | 问答结果审核 | 检查系统回答是否答非所问、缺引用、过度承诺 |
| CHUNKREVIEW | 文献片段复核 | 检查文献片段标题、年份、内容是否明显错误 |

老师或负责人会告诉你具体做哪一个任务。不要同时乱填多个任务文件。

---

## 你不需要做什么

请不要做以下事情：

- 不需要写代码。
- 不需要安装软件。
- 不需要连接服务器。
- 不需要操作数据库。
- 不需要判断某种干预是否真的有效。
- 不需要写正式医学结论。
- 不需要改文件表头。
- 不需要合并多个同学的结果。

如果你不确定，写在 `notes` 字段里即可。

---

## 文件怎么填

一般会给你一个 `.csv` 模板文件。可以用 Excel、WPS、LibreOffice 或文本编辑器打开。

填写规则：

1. 不要删除表头。
2. 每一行是一条记录。
3. 多个关键词或别名用英文分号 `;` 分隔。
4. 布尔字段只填 `true`、`false` 或 `uncertain`。
5. 不确定的内容不要硬猜，可以写 `uncertain` 或在 `notes` 里说明。
6. 中文内容不要写成乱码。
7. 交付前检查文件能正常打开。

英文分号示例：

```text
ADOS;自闭症;诊断
```

不要用中文顿号、逗号混写：

```text
ADOS、自闭症，诊断
```

---

## 文件怎么命名

完成后的结果文件按下面格式命名：

```text
<任务编号>_<你的学生编号>_<你的姓名拼音>_result.csv
```

示例：

```text
QAQUESTION_S01_zhangsan_result.csv
SAFETYQUESTION_S02_lisi_result.csv
ALIAS_S03_wangwu_result.csv
```

如果有额外说明，可以再交一个 notes 文件：

```text
QAQUESTION_S01_zhangsan_notes.md
```

---

## 文件怎么返还

把结果文件放进一个文件夹，压缩成 zip。

压缩包命名：

```text
asd_kgrag_return_<任务编号>_<学生编号>_<姓名拼音>_<日期>.zip
```

示例：

```text
asd_kgrag_return_QAQUESTION_S01_zhangsan_20260610.zip
```

返还压缩包里建议包含：

```text
QAQUESTION_S01_zhangsan_result.csv
QAQUESTION_S01_zhangsan_notes.md
```

如果没有额外说明，`notes.md` 可以不交。

---

## 提交前自查

提交前请检查：

```text
1. 文件名是否符合要求？
2. student_id 是否每行都填了？
3. 必填字段有没有空着？
4. CSV 是否能正常打开？
5. 多个关键词/别名是否用英文分号 ; 分隔？
6. true/false 字段有没有乱填？
7. 是否有大段复制重复内容？
8. 不确定的地方是否写到了 notes？
```

---

## 质量要求

项目不要求你们给出完美答案，但要求结果清楚、可检查。

好的结果：

- 问题自然，不是关键词堆砌。
- 分类基本合理。
- 关键词能帮助别人理解问题。
- 不确定的地方明确写出来。
- 文件格式稳定。

不好的结果：

- 复制粘贴大量重复问题。
- 空很多必填字段。
- 文件名乱写。
- 把多个问题塞进同一格。
- 自己改表头。
- 把不确定的内容写成肯定结论。

---

## 遇到不会填怎么办

优先看对应任务说明：

- `QAQUESTION_student_task.md`
- `SAFETYQUESTION_student_task.md`
- `ALIAS_student_task.md`
- `QAREVIEW_student_task.md`
- `CHUNKREVIEW_student_task.md`

仍然不确定，就在 `notes` 字段里说明。

不要为了填满而乱编。
